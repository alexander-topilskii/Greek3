# Инструкции для AI-агентов

## Перед началом работы

1. Прочитай `agents/PROJECT.md` — цели и структура
2. Прочитай `agents/ARCHITECTURE.md` — как устроена сборка
3. При работе с контентом — `agents/MD_FORMAT.md`
4. При работе через Cloud Agent / PR — `agents/PR_WORKFLOW.md`

## Правила

- **Контент** — только в `words/**/*.md`, не хардкодить слова в HTML/JS
- **Уроки** — в `words/lessons/` хранятся только **ссылки** на слова; сами MD-файлы слов — всегда в тематической категории (`verbs/`, `nouns/` и т.д.). При добавлении слова в урок **обязательно** создать или обновить файл в подходящей категории и ссылку в её `readme.md` (см. `agents/MD_FORMAT.md` → «Уроки»). **В урок добавлять только если пользователь явно написал «в урок»** — иначе достаточно разделов словаря (`nouns/`, `verbs/` и т.д.)
- **Стили** — mobile-first, светлая тема, анимации через CSS transitions/keyframes
- **Сборка** — изменения в генераторе: `src/build/`; статика: `site/`
- **Не коммитить** `dist/` — генерируется в CI
- **Язык UI** — русский; греческий текст — в Unicode (не translit)
- **Pull request** — ветка `cursor/<описание>-b6d3`, PR не в draft; после зелёного CI auto-merge сработает сам (см. `agents/PR_WORKFLOW.md`). **Не мержить PR вручную** — только создавать и обновлять

## Типичные задачи

| Задача | Где менять |
|--------|------------|
| Добавить слово | `words/<кат>/<имя>.md` + ссылка в `readme.md` |
| Добавить слово в урок | 1) файл слова в `words/<кат>/` + ссылка в `words/<кат>/readme.md`; 2) ссылка в `words/lessons/<NN>/readme.md`. Категорию подобрать по типу слова; если подходящей нет — создать новую (см. MD_FORMAT) |
| Новый раздел | Папка в `words/` + `readme.md` + ссылка в `words/readme.md` и `Readme.md`; карточка на главной — `homeSections` в `src/build/index.ts` |
| Стили/анимации | `site/css/main.css` |
| Интерактив практики | `site/js/practice.js` |
| Шаблон страницы | `src/build/render.ts` |
| Парсинг MD | `src/build/parse-word.ts` |
| CI/CD | `.github/workflows/deploy.yml` |

## Проверка локально

```bash
npm run build:site && npm run dev
# Открыть http://localhost:3000
```

## Обновление этой документации

Если узнал что-то новое о проекте — **добавь в соответствующий файл в `agents/`**, не только в ответ пользователю.
