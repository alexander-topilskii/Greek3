# Формат Markdown-файлов

## Оглавление раздела (`readme.md`)

Файлы `words/**/readme.md` — навигационные страницы. Поддерживается обычный Markdown со ссылками на слова.

Заголовки `## Название группы` группируют слова на странице раздела:

```markdown
# Прилагательные

Краткое описание раздела (необязательно).

## Цвета

[άσπρος — белый](белый άσπρος.md)
[κόκκινος — красный](красный κόκκινος.md)
```

При сборке ссылки превращаются в `.html`, группы отображаются с подзаголовками.

## Метаданные (frontmatter)

```yaml
---
level: A2
topics: [Еда и напитки]
tags: [исчисляемое]
type: phrase
---
```

- `level` — A1, A2, B1, B2 (если нет — по категории)
- `topics` — темы; если нет — из группы `##` в readme категории
- `type` — `verb`, `noun`, `phrase` и т.д.

При сборке генерируются страницы **Тем** и **Уровней**; на карточках списка — бейджи.

## Файл слова

Имя файла = slug страницы в формате `{русский} {греческое_настоящее}.md` (например `verbs/ухожу φεύγω.md` → `/words/verbs/ухожу φεύγω.html`). При нескольких переводах через `/` в секции «База» в имени файла берётся первый вариант до `/`.

### Обязательные секции

```markdown
# База
перевод/синоним : форма1 - форма2 - форма3

# Формы
ελληνικά - перевод
ελληνικά - перевод
```

### Секция «База»

Одна строка после заголовка:

```
русский_перевод : прошедшее - настоящее - будущее
```

Пример:

```
ухожу/уезжаю : έφυγα - φεύγω - θα φύγω
```

Парсер извлекает:
- `translation` — часть до `:`
- `forms` — массив греческих форм через `-`

### Секция «Формы»

Каждая строка: `греческий_текст - русский_перевод`

```
έφυγα - я ушёл
φεύγω - я ухожу
```

Пустые строки игнорируются.

### Дополнительные секции

Любые другие `# Заголовок` секции сохраняются как произвольный контент и отображаются на странице слова. Поддерживается **Markdown**: жирный текст, списки, таблицы.

Секция `# Тип` — группа глагола (Τύπος Α, Τύπος Β1 + Β2, неправильные) или тип записи (πτώση, падеж).

Секция `# Контекст` — примеры: `- **греческий** — перевод` (для фраз).

## Фразы (`words/phrases/`)

```markdown
# База
счёт, пожалуйста : Τον λογαριασμό, παρακαλώ

# Формы
Τον - (артикль)
λογαριασμό - счёт
παρακαλώ - пожалуйста
Τον λογαριασμό, παρακαλώ - счёт, пожалуйста

# Контекст
- **Τον λογαριασμό, παρακαλώ.** — Счёт, пожалуйста.
```

### Правило для агента при добавлении фразы

1. В секции `# Формы` перечислить **перевод каждого слова** фразы (`греч. - рус.`) в порядке появления; в конце — строка с полной фразой (и варианты, если есть).
2. **Каждое новое слово**, которого ещё нет в словаре (`words/<кат>/`), добавить отдельным MD-файлом и ссылкой в `readme.md` категории. Формы глаголов, местоимения и т.п., уже представленные в словаре, **не дублировать**.
3. Категорию слова подбирать по типу (глагол → `verbs/`, существительное → `nouns/`, наречие/местоимение → `pronouns/`, частица → `particles/`).

Минимальный набор форм — прошедшее, настоящее, будущее; полные спряжения добавляются по мере необходимости (см. `ухожу φεύγω.md`).

## Категории

Текущая структура:

```
words/
├── readme.md          # → words/index.html (оглавление словаря)
├── lessons/           # → уроки (только ссылки на слова, не дублировать контент)
│   ├── readme.md
│   └── 01/readme.md … 50/readme.md
├── verbs/
├── nouns/
├── adjectives/
├── pronouns/
├── numbers/
├── cases/
├── particles/
└── phrases/
    ├── readme.md
    └── *.md
```

Страницы `topics/` и `levels/` генерируются при сборке (исходники — только `readme.md`-хабы).

## Уроки (`words/lessons/`)

Урок — список слов с занятия с репетитором. **Файлы слов не создаются внутри `lessons/`** — только ссылки на MD в тематических категориях.

**Важно:** ссылки в `words/lessons/<NN>/readme.md` добавлять **только если пользователь явно попросил** («в урок», «добавь в урок N» и т.п.). Запрос «добавить в разделы» / «добавить слова» без упоминания урока — **не** создавать и не дополнять уроки.

### Алгоритм для агента при добавлении слова в урок

1. **Определить категорию** по типу слова:
   - глагол → `verbs/`
   - существительное → `nouns/`
   - прилагательное → `adjectives/`
   - местоимение, наречие места/времени → `pronouns/`
   - числительное → `numbers/`
   - частица, союз, связка для письма → `particles/`
   - устойчивая фраза → `phrases/`
   - грамматика падежа → `cases/` (обычно не добавляют в уроки)
2. **Если подходящей категории нет** — создать новую:
   - папка `words/<кат>/`
   - `words/<кат>/readme.md` с заголовком и описанием
   - ссылка в `words/readme.md`
   - запись в `CATEGORY_LABELS` и `homeSections` в `src/build/index.ts`
3. **Создать или обновить файл слова** в категории: `words/<кат>/{рус} {греч}.md` (формат см. выше).
4. **Добавить ссылку в `readme.md` категории** — в подходящую группу `## …` или новую группу.
5. **Добавить ссылку в файл урока** `words/lessons/<NN>/readme.md`, секция `## Слова`:

```markdown
[дом](../../nouns/дом σπίτι.md)
```

Путь относительный от папки урока (`01/`, `02/`, …) к файлу в категории.

### Чего не делать

- Не класть полный контент слова (секции «База», «Формы») только в урок
- Не дублировать один и тот же MD в `lessons/` и в категории
- Не добавлять в intro урока примеры в синтаксисе `[текст](url.md)` — парсер воспримет их как реальные ссылки
- **Не добавлять слова в урок без явной просьбы пользователя** — даже если слова пришли с занятия
