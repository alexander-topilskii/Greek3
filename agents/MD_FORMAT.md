# Формат Markdown-файлов

## Оглавление раздела (`readme.md`)

Файлы `words/**/readme.md` — навигационные страницы. Поддерживается обычный Markdown со ссылками на слова.

Заголовки `## Название группы` группируют слова на странице раздела:

```markdown
# Прилагательные

Краткое описание раздела (необязательно).

## Цвета

[άσπρος — белый](белый άσπρος.md)
[κόκκινος — красный](красный κόκκινος.md)
```

При сборке ссылки превращаются в `.html`, группы отображаются с подзаголовками.

## Файл слова

Имя файла = slug страницы в формате `{русский} {греческое_настоящее}.md` (например `verbs/ухожу φεύγω.md` → `/words/verbs/ухожу φεύγω.html`). При нескольких переводах через `/` в секции «База» в имени файла берётся первый вариант до `/`.

### Обязательные секции

```markdown
# База
перевод/синоним : форма1 - форма2 - форма3

# Формы
ελληνικά - перевод
ελληνικά - перевод
```

### Секция «База»

Одна строка после заголовка:

```
русский_перевод : прошедшее - настоящее - будущее
```

Пример:

```
ухожу/уезжаю : έφυγα - φεύγω - θα φύγω
```

Парсер извлекает:
- `translation` — часть до `:`
- `forms` — массив греческих форм через `-`

### Секция «Формы»

Каждая строка: `греческий_текст - русский_перевод`

```
έφυγα - я ушёл
φεύγω - я ухожу
```

Пустые строки игнорируются.

### Дополнительные секции

Любые другие `# Заголовок` секции сохраняются как произвольный контент и отображаются на странице слова.

Секция `# Тип` — группа глагола (Τύπος Α, Τύπος Β1 + Β2, неправильные).

Минимальный набор форм — прошедшее, настоящее, будущее; полные спряжения добавляются по мере необходимости (см. `ухожу φεύγω.md`).

## Категории

Текущая структура:

```
words/
├── readme.md          # → words/index.html (оглавление словаря)
├── verbs/
├── nouns/
├── adjectives/
├── pronouns/
├── numbers/
├── cases/
└── particles/
    ├── readme.md      # → words/<категория>/index.html
    └── *.md           # → words/<категория>/<имя>.html
```

Планируются: `cases/` (падежи), `particles/` (частицы) — см. корневой `Readme.md`.
